# 中文文章与视频学习资料

最后核验日期：2026 年 6 月 9 日

## 使用说明

- 每个阶段先看一篇文章，再看一个视频，然后立即完成项目任务；
- `必学`内容需要完整学习，`选学`内容只在遇到问题时查询；
- B 站视频可能被作者删除或改名，因此同时记录标题和 BV 号；
- 如果视频链接失效，在 B 站搜索完整标题或 BV 号；
- Garmin 开发中文资料较少，使用浏览器翻译阅读官方英文文档；
- 不连续看两小时视频。单次最多 40 分钟，之后必须动手实践。

## 第 3 阶段：领域知识、混合检索与 RAG

对应时间：第 3 至 4 周

- `选学` [NVIDIA：什么是 RAG](https://www.nvidia.cn/glossary/retrieval-augmented-generation/)  
  中文概念文章，适合复习。

### 视频

- `必学` [什么是 RAG 检索增强生成](https://www.bilibili.com/video/BV1cb421p7UQ/)  
  BV号：`BV1cb421p7UQ`。先理解检索、增强和生成。
- `选学` [RAG 优化策略与评测体系](https://www.bilibili.com/video/BV1LJ4m1T7wc/)  
  BV号：`BV1LJ4m1T7wc`。内容较长，按项目问题选择观看。

### 项目特别要求

本项目不能把所有知识都放进向量库：

- 设备能力、组件状态和认证变体：结构化 JSON 查询；
- 硬性限制：规则引擎；
- 设计案例和风格描述：语义检索；
- 用户选择与拒绝：反馈数据统计；
- Agent：综合证据并记录 `DecisionTrace`。

### 完成标准

- 能解释为什么设备 API 不能只靠语义检索；
- 能实现组件、状态、风格变体和规则的混合查询。

## 链接失效处理

发现链接失效时：

1. 在 B 站使用完整标题或 BV 号搜索；
2. 官方文档从网站首页按文章标题搜索；
3. 不使用来源不明的网盘资料；
4. 在每日笔记记录失效链接；
5. 将替代链接更新到本文件，并修改“最后核验日期”。
