# 学习导航

## 先明确现实目标

每天 2 小时、约 3 个月，总学习时间约 180 小时。这足以让你：

- 理解大模型和 Agent 的核心概念；
- 独立开发一个有前后端、RAG、工具调用和评测的 Agent；
- 能针对简单业务场景完成需求分析、方案选型和演示；
- 形成一套可以用于求职或内部转岗的作品集。

但 3 个月通常不足以直接成为成熟的“专家”或大型团队负责人。更现实的阶段目标是：

> 成为具备完整项目作品、能独立负责小型 AI Agent 项目的候选人，并建立继续成长为负责人的正确能力结构。

## 你真正需要掌握的六类能力

1. **模型认知**：模型能力边界、Token、上下文、幻觉、结构化输出。
2. **应用开发**：Node.js、模型 API、流式输出、会话管理、前后端。
3. **Agent 设计**：任务拆解、工具调用、记忆、工作流、MCP。
4. **知识增强**：文档切分、检索、向量、重排、引用与 RAG。
5. **质量工程**：评测集、成功率、成本、延迟、日志、安全和回退。
6. **负责人能力**：业务价值、技术选型、排期、风险、复盘和沟通。

## 每天固定节奏

每天 2 小时建议这样分配：

| 时间 | 活动 |
| --- | --- |
| 20 分钟 | 回顾昨天内容，写下今天要解决的一个问题 |
| 40 分钟 | 阅读文档或观看课程，只学今天需要的内容 |
| 50 分钟 | 动手编码、实验或完善项目 |
| 10 分钟 | 写笔记：结论、证据、失败原因、明日任务 |

学习时禁止连续看 2 小时视频。没有输出的观看很难转化成能力。

## 每周固定产出

- 5 篇以上每日笔记；
- 1 次可运行演示；
- 1 份每周复盘；
- 1 个失败案例及修复记录；
- 更新一次项目 README 和进度表。

## 判断是否真的学会

对每个主题，必须能够不用照抄资料完成以下四件事：

1. 用中文解释它解决什么问题。
2. 画出输入、处理、输出流程。
3. 写出最小可运行示例。
4. 说明它会在哪些情况下失败，以及如何验证。
