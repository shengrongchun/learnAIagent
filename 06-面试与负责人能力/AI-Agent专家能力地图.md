# AI Agent 专家能力地图

“专家”不是会使用最多框架，而是面对新问题时，能稳定做出正确判断并用证据改进系统。

## 五个成长层级

| 层级 | 能力表现 | 在 Garmin 项目中的证据 |
| --- | --- | --- |
| L1 会调用 | 能调用模型并生成结果 | 中文需求生成 Monkey C 代码 |
| L2 会构建 | 能组合 RAG、工具和工作流 | 查询规则、写代码、调用编译器 |
| L3 会保证质量 | 有评测、安全、日志和回退 | 评测集、编译闭环、越权限制 |
| L4 会做架构判断 | 能权衡模型、框架、成本和复杂度 | 手写与框架对比、架构决策记录 |
| L5 会负责结果 | 能定义业务指标、带领迭代并迁移方法 | 项目复盘、答辩、其他领域方案 |

三个月的目标是稳定达到 L3，并开始具备 L4 和 L5 的思考方式。

## 专家的八项核心能力

### 1. 问题定义

- 判断问题是否适合 Agent；
- 找到真实用户、任务频率、错误代价和成功指标；
- 拒绝没有业务价值的“为了 AI 而 AI”。

### 2. 模型与上下文工程

- 理解模型边界、上下文和结构化输出；
- 知道 Prompt、RAG、工具和微调分别解决什么问题；
- 能减少不必要上下文和模型调用。

### 3. 领域建模与知识工程

- 将领域知识拆成结构化事实、规则、文档、案例和反馈；
- 知道何时使用精确查询、规则引擎或语义检索；
- 让每项 Agent 决策能够追溯到证据；
- 建立知识更新、版本和单独评测机制。

### 4. Agent 架构

- 在固定工作流、单 Agent、多 Agent 中正确选择；
- 将复杂能力拆成可测试 Skills，并决定由工作流还是 Agent 编排；
- 设计状态、结束条件、回退和人工接管；
- 控制循环、成本和不确定性。

### 5. 持续学习与知识运营

- 区分模型训练、知识更新、反馈学习和规则演进；
- 建立生产循环与离线学习循环；
- 将反馈转化为假设、实验、评测和认证知识；
- 防止 Agent 根据噪声反馈或单次结果自我污染。

### 6. 工具与系统集成

- 为工具设计清晰参数、权限、超时和错误；
- 将不稳定模型与确定性系统可靠连接；
- 理解 MCP 的价值和适用边界。

### 7. 评测

- 在开发前定义评测集和指标；
- 区分模型问题、检索问题、工具问题和流程问题；
- 每次修改都运行回归测试。

### 8. 安全与可观测性

- 防止提示词注入、越权和数据泄露；
- 使用日志、Trace、告警和预算理解系统；
- 设计安全退出、降级和人工接管。

### 9. 产品与商业判断

- 用任务成功率、节省时间和成本衡量价值；
- 管理范围和优先级；
- 能向非技术人员清楚说明收益与风险。

### 10. 迁移与带领能力

- 将一个项目的方法迁移到新领域；
- 写清架构决策和复盘，帮助团队复用经验；
- 能审查方案、发现风险并指导改进。

## 每月证据

### 第 1 个月

- 能手写模型调用、结构化输出和最小代码生成；
- 有至少 20 条测试需求；
- 能解释 Agent 与工作流的区别。

### 第 2 个月

- 有 RAG、工具调用、自动编译和修复闭环；
- 有失败分类和至少 30 条评测数据；
- 能说明是否使用框架和 MCP 的依据。

### 第 3 个月

- 有安全、日志、成本和批量任务控制；
- 有完整复盘、架构决策和演示；
- 能独立设计另一个领域的 Agent 方案。
