# Skills 设计清单

## Skill 与 Agent 的区别

- **Skill**：边界明确、可复用、可单独测试的能力，例如“分析设计覆盖缺口”。
- **Agent**：围绕目标规划步骤，选择并调用多个 Skills，根据结果决定下一步。
- **Workflow**：步骤和分支较确定的流程，例如“变体认证流水线”。

## 生产生成 Skills

| Skill | 输入 | 输出 |
| --- | --- | --- |
| 需求解析 | 中文需求 | `DesignIntent` |
| 需求澄清 | 缺失或冲突信息 | 澄清问题与更新需求 |
| 设备查询 | 设备 ID | 设备能力 |
| 组件选择 | 需求、设备 | 可行组件与评分 |
| 变体检索 | 组件、风格、设备 | 认证组件变体 |
| 候选差异规划 | 需求、变体、布局 | 候选差异计划 |
| 规格生成 | 决策证据 | `WatchFaceSpec` |
| 规则检查 | `WatchFaceSpec` | 问题清单 |
| 预览渲染 | `WatchFaceSpec` | 多状态预览 |
| Monkey C 生成 | 确认规格 | Monkey C |
| 编译与测试 | Monkey C、设备 | 构建结果 |
| 错误修复 | 错误、规格、代码 | 修复后的代码 |

## Skill 质量要求

每个 Skill 必须有：

- 明确输入和输出 Schema；
- 适用范围与不支持范围；
- 可用工具和权限；
- 错误分类和失败输出；
- 测试样例；
- 评测指标；
- 版本；
- 日志与可追溯信息。

如果一个能力无法独立测试，通常说明它的边界还不清楚，不适合直接交给 Agent 调用。
